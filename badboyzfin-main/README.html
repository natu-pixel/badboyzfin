Jellyfin Theme

badboyzfin

A dark glassmorphism theme for Jellyfin. Cards lift, dialogs breathe, the sidebar stays put.

v2 — FIXED

// What's fixed in v2

  • Cards (movies, series, channels) now open on click — will-change removed from .cardBox which was creating a stacking context that swallowed pointer events in Chromium & WebKit
  • All card children explicitly get pointer-events: auto — overlay buttons, links, and scaler elements are all clickable
  • Pseudo-elements (::before / ::after) set to pointer-events: none — decorative glows can no longer intercept taps
  • smooth-transitions: replaced transition: all on buttons with specific properties — was accidentally transitioning pointer-events
  • smooth-transitions: page container no longer animates transform on load — was creating a stacking context mid-navigation
Installation
01 /

Open Dashboard

Go to Admin → Dashboard → General in your Jellyfin instance.

02 /

Custom CSS field

Scroll to the Custom CSS text area at the bottom of the General settings page.

03 /

Paste & Save

Paste the single @import line below. Click Save. Hard-refresh your browser.

04 /

Self-host (optional)

Drop the theme/ folder on your server. Point the @import at your own URL for full control.

Custom CSS — paste this 
/* badboyzfin v2 — complete theme with click fix */
@import url("https://cdn.jsdelivr.net/gh/JamsRepos/badboyzfin@latest/theme/base.css");
@import url("https://cdn.jsdelivr.net/gh/JamsRepos/badboyzfin@latest/theme/modules/static-sidebar.css");
@import url("https://cdn.jsdelivr.net/gh/JamsRepos/badboyzfin@latest/theme/modules/moving-cards.css");
@import url("https://cdn.jsdelivr.net/gh/JamsRepos/badboyzfin@latest/theme/modules/smooth-transitions.css");

/* ── Card click fix (always include this) ── */
.cardBox, .cardBox *, .card, .cardLink,
.cardScalable, .cardOverlayContainer,
.cardOverlayButton, .cardImageContainer { pointer-events: auto !important; }
.cardBox::before, .cardBox::after,
.cardBox *::before, .cardBox *::after { pointer-events: none !important; }
M3U / IPTV — why nothing shows up

M3U Tuner checklist

  • Use the raw file URL or absolute file path — Jellyfin can't read relative paths
  • Path example: /config/iptv/playlist.m3u
  • Remote URL must be reachable from your server, not just your browser
  • Each stream line needs #EXTINF:-1 before the URL
  • Add group-title="Sports" in the EXTINF tag for categories
  • After adding, go to Dashboard → Live TV and click Refresh Guide

XMLTV EPG checklist

  • File path or URL must point to a valid .xml or .xmltv file
  • Channel IDs in the XMLTV must match channel IDs in your M3U (the tvg-id attribute)
  • Example EXTINF: #EXTINF:-1 tvg-id="bbc1" tvg-name="BBC One"
  • After saving, run the Refresh Guide scheduled task manually
  • EPG data can take 5–10 minutes to populate after refresh
  • Check Dashboard → Scheduled Tasks for errors
⚠ Most common reason nothing shows up The M3U path is correct but the stream URLs inside it use a hostname that your Jellyfin server can't resolve (e.g. localhost pointing to your PC instead of the IPTV provider). Open one stream URL in a browser on the server machine to verify it works. Also make sure your IPTV provider hasn't expired your subscription.
